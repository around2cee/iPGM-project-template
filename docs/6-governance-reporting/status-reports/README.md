# Status Reports
> iPGM 6.2 — Weekly / Monthly Program Status Reports

Store dated status report files in this folder.

## Naming Convention
```
YYYY-MM-DD-status-report.md      (weekly)
YYYY-MM-monthly-status-report.md (monthly)
```

## What Each Status Report Should Cover
- Reporting period
- Overall program status (Green / Yellow / Red)
- Accomplishments this period
- Changes to plan since last update
- Goals for next period
- Active risks & issues
- Next key milestone

Use the **Status Report** GitHub Issue Template to capture and track status updates as issues,
or create markdown files here for document-based reporting.
